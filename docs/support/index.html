Support

Whir shows the API-equivalent cost of your local Claude Code and Codex usage, in your Mac menu bar. Everything is computed on your machine — no account, nothing uploaded (the only network request is an optional daily refresh of its price table).

The menu-bar number is $0 / looks empty

The headline is today's spend, so it starts at $0 each morning and grows as you use Claude Code / Codex. Open the popover to see your last-30-days value and the per-provider breakdown.

It says "No logs found"

Whir reads ~/.claude/projects and ~/.codex/sessions. If you haven't used those tools yet, or installed them elsewhere, there's nothing to read. In the Mac App Store build, grant access to each folder once when prompted (press ⇧⌘. to reveal hidden folders in the picker).

The numbers are estimates

Whir multiplies token counts by published API prices (stamped with an "as of" date, refreshed daily from GitHub unless you turn that off). Subscription quotas aren't token-denominated, so this is usage value, not your actual bill. A model missing from the price table shows — instead of a cost.

Launch at login

Toggle it in the settings window (gear icon). If macOS shows it as needing approval, enable Whir under System Settings → General → Login Items.
